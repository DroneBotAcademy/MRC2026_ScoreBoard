# 🏁 Drone Racing Dashboard

ระบบจัดการเวลาและตารางอันดับการแข่งขันโดรน (Drone Racing Management System) ที่เชื่อมต่อกับ Google Sheets เป็นฐานข้อมูลแบบ Real-time

## 💡 หลักการทำงานของระบบ (System Overview)

ระบบนี้ถูกออกแบบมาเพื่อใช้ในการจับเวลาและบันทึกผลการแข่งขันโดรน โดยแบ่งการทำงานออกเป็น 2 ส่วนหลัก:
1.  **Frontend (Web Dashboard):** พัฒนาด้วย HTML, CSS และ JavaScript ทำหน้าที่แสดงผลหน้าจอการจับเวลา, รายชื่อทีม และตารางอันดับ (Leaderboard)
2.  **Backend (Google Sheets API):** ใช้ Google Apps Script เป็นตัวกลางในการรับ-ส่งข้อมูลระหว่างหน้าเว็บกับ Google Sheets เพื่อจัดเก็บข้อมูลทีมและสถิติเวลาแข่ง

**Workflow การทำงาน:**
- เมื่อเปิดหน้าเว็บ ระบบจะดึงข้อมูลการตั้งค่าและธีมจาก `localStorage`
- เมื่อเลือกรายการแข่งขัน (Air / Remote) ระบบจะดึงข้อมูลทีมจาก Google Sheets มาเก็บไว้ในตัวแปร Global (`allTeamsData`)
- ผู้ควบคุมสามารถเลือกทีม เริ่มจับเวลา และบันทึกเวลา (Lap) ลงระบบได้ทันที
- ข้อมูลที่บันทึกจะถูกส่งไปเก็บใน Google Sheets และอัปเดตหน้า Leaderboard แบบอัตโนมัติ

## 🛠️ รายละเอียดฟังก์ชันและหน้าการทำงาน

### 1. หน้าหลัก (Home Page)
- **`enterEvent(eventName)`**: ทำหน้าที่เปลี่ยนจากหน้าแรกเข้าสู่หน้า Dashboard ของรายการที่เลือก (เช่น Air Circuit หรือ Remote Pro) พร้อมสั่งโหลดข้อมูลจากฐานข้อมูลตามหมวดหมู่ที่เลือก
- **`exitToHome()`**: กลับสู่หน้าหลักและรีเซ็ตสถานะการแสดงผล

### 2. ระบบจัดการข้อมูล (Database & Sync)
- **`syncWithDatabase(category)`**: ฟังก์ชันสำคัญที่ใช้ดึงข้อมูล (GET) จาก Google Apps Script โดยส่ง Parameter เป็นชื่อหมวดหมู่ เพื่อนำข้อมูลรายชื่อทีมและสถิติเดิมมาแสดงผล
- **`addLap()`**: ฟังก์ชันสำหรับบันทึกเวลา (POST) เมื่อสิ้นสุดการแข่งขันในแต่ละรอบ โดยจะส่งข้อมูล ID ทีม, รอบที่แข่ง, และเวลาที่ทำได้ไปยัง Google Sheets

### 3. ระบบจับเวลา (Timer Functions)
- **`startTimer()` / `stopTimer()`**: ควบคุมการเริ่มและหยุดตัวเลขจับเวลา
- **`resetTimer()`**: ล้างค่าเวลาให้เป็น 0 เพื่อเริ่มใหม่
- **`updateTimerDisplay()`**: อัปเดตตัวเลขเวลาที่หน้าจอแบบ Milliseconds ให้เป็นปัจจุบัน

### 4. การแสดงผลข้อมูล (Data Rendering)
- **`renderSingleTeam()`**: แสดงรายละเอียดของทีมที่กำลังเลือกอยู่ รวมถึงประวัติการวิ่ง (Laps) ของทีมนั้นๆ
- **`renderLeaderboard()`**: ประมวลผลข้อมูลจากทุกทีม เรียงลำดับตามเวลาที่ดีที่สุด และแสดงผลในรูปแบบตารางอันดับ (Ranking)
- **`switchTab(value)`**: สลับโหมดการดูระหว่าง "ดูเฉพาะทีมที่เลือก" กับ "ดูตารางอันดับทั้งหมด"

### 5. ระบบเสริมและการจัดการข้อผิดพลาด
- **`toggleTheme()`**: สลับโหมด Dark/Light และบันทึกค่าลงในเครื่อง (Local Storage)
- **Error Handling**: มีระบบตรวจจับสถานะอินเทอร์เน็ต (Online/Offline) และแจ้งเตือนเมื่อการเชื่อมต่อฐานข้อมูลมีปัญหา

## 📊 โครงสร้างข้อมูล (Data Structure)

ข้อมูลจะถูกจัดเก็บในรูปแบบ Object ดังนี้:
```javascript
allTeamsData = {
    "team_id": {
        name: "ชื่อทีม",
        laps: [
            { roundNum: 1, timeStr: "0:45:230" },
            { roundNum: 2, timeStr: "0:43:120" }
        ]
    }
}
```

## 🚀 วิธีการติดตั้งและใช้งาน

1.  **เตรียมฐานข้อมูล:**
    - สร้าง Google Sheets และตั้งชื่อคอลัมน์ให้ตรงตามที่ระบบกำหนด (`id`, `name`, `laps`, `category`)
    - Deploy Google Apps Script และคัดลอก Web App URL
2.  **ตั้งค่าโค้ด:**
    - นำ URL ที่ได้ไปใส่ในตัวแปร `SCRIPT_URL` ในไฟล์ `app.js`
3.  **ใช้งาน:**
    - เปิดไฟล์ `index.html` ผ่าน Browser หรือ Web Server

## 🚨 ตัวชี้วัดสถานะระบบ

| สถานะ | ความหมาย |
| :--- | :--- |
| **Sync Success** | เชื่อมต่อฐานข้อมูลสำเร็จ ข้อมูลเป็นปัจจุบัน |
| **Connection Lost** | อินเทอร์เน็ตขาดการติดต่อ ระบบจะพยายามเชื่อมต่อใหม่ |
| **No Teams Found** | ไม่พบข้อมูลทีมในหมวดหมู่ที่เลือก โปรดตรวจสอบใน Google Sheets |
